## Безопасный шаблон переменных окружения

20 июля 2026 года завершён дополнительный аудит секретов проекта.

Подтверждено:

- файл `.env` не отслеживается Git;
- файл `.env` имеет права доступа `600`;
- резервные пароли в Playwright worker-файлах отсутствуют;
- `POSTGRES_PASSWORD` является обязательной переменной;
- при отсутствии `POSTGRES_PASSWORD` worker завершает работу с понятной ошибкой;
- явные пароли и токены в `docker-compose.yml` отсутствуют;
- устаревший файл `.env.save` удалён;
- создан безопасный файл `.env.example` без настоящих секретов.

Старые упоминания тестового пароля обнаружены в истории Git. История репозитория пока не переписывалась, поскольку действующий пароль PostgreSQL уже был изменён.

## Аудит доступа к n8n и ротация PostgreSQL — 2026-07-20

- n8n использует встроенную пользовательскую аутентификацию по email.
- Первоначальная учётная запись владельца n8n уже настроена.
- Порт n8n привязан только к `127.0.0.1:5678`.
- Порт 5678 отсутствует в разрешающих правилах UFW.
- Reverse proxy и HTTPS пока не настроены.
- `N8N_SECURE_COOKIE=false` используется временно до перехода на HTTPS.
- Устаревшие переменные Basic Auth удалены из `.env`.
- Пароль PostgreSQL повторно ротирован после появления предыдущего значения в диагностическом выводе.
- Проверено, что новый пароль работает, а предыдущий пароль отклоняется.
- В `.env` исправлено имя базы с `competitor_db` на фактическое `competitor_analyzer`.
- Защищённая временная копия `.env` со старым паролем удалена.
- Аудит n8n выявил SQL-ноды без Query Parameters, неиспользуемые credentials и Code nodes, требующие отдельной проверки.
