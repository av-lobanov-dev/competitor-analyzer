# ADR-001. Разделение ответственности компонентов

- Статус: принято
- Дата: 2026-07-15

## Контекст

В проекте используются n8n, Playwright, PostgreSQL и GPT.

Без чёткого разделения ответственности логика может дублироваться в разных компонентах. Это усложнит разработку, тестирование и поиск ошибок.

## Решение

Ответственность компонентов разделяется следующим образом:

### n8n

Управляет процессами:

- запускает workflow;
- управляет расписанием;
- создаёт задания;
- передаёт данные;
- запускает уведомления.

### Playwright

Выполняет работу с сайтами:

- открывает страницы;
- переходит по ссылкам;
- выполняет JavaScript;
- собирает структуру страниц;
- собирает товары и цены.

### PostgreSQL

Хранит:

- задания;
- снимки страниц;
- правила парсинга;
- товары;
- цены;
- историю изменений;
- результаты анализа.

### GPT

Выполняет анализ:

- изучает структуру сайта;
- создаёт правила парсинга;
- сопоставляет товары;
- анализирует цены;
- подготавливает отчёты.

## Причины

Такое разделение:

- уменьшает связанность компонентов;
- упрощает тестирование;
- упрощает поиск ошибок;
- позволяет независимо масштабировать компоненты;
- делает архитектуру понятной.

## Последствия

Playwright не должен принимать бизнес-решения.

GPT не должен непосредственно обходить сайты.

n8n не должен использоваться как основное хранилище бизнес-данных.

PostgreSQL становится основным источником истины проекта.
