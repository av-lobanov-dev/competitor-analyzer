# ADR-003. Документация хранится в Git

- Статус: принято
- Дата: 2026-07-20

## Контекст

Информация о проекте была распределена между несколькими чатами.

Это затрудняло:

- поиск актуальной информации;
- понимание текущего состояния проекта;
- восстановление принятых решений;
- передачу проекта другому разработчику;
- сопоставление документации с версией кода.

## Решение

Создать техническую энциклопедию проекта в каталоге:

```text
docs/
```

Каталог располагается в основном Git-репозитории проекта.

Git является основным источником технической документации.

## Причины

Документация в Git:

- находится рядом с кодом;
- имеет историю изменений;
- может обновляться вместе с кодом;
- доступна на сервере и в GitHub;
- может быть восстановлена из предыдущей версии;
- удобно читается в формате Markdown.

## Дополнительные инструменты

Google Таблицы могут использоваться для:

- списка клиентов;
- рабочих реестров;
- коммерческих расчётов;
- ручной проверки товаров;
- задач, которые редактируют сотрудники без доступа к Git.

Google Таблицы не являются основным источником технической архитектуры.

## Последствия

После значимого изменения проекта необходимо обновлять соответствующие документы.

Чаты используются для обсуждения и разработки решений.

Утверждённая информация переносится в каталог `docs`.
