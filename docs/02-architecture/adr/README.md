# Архитектурные решения

ADR — Architecture Decision Record.

Это отдельный документ, в котором фиксируется важное архитектурное решение проекта.

## Когда нужно создавать ADR

ADR создаётся, если принимается решение, которое:

- влияет на архитектуру проекта;
- определяет ответственность компонентов;
- влияет на структуру базы данных;
- определяет способ обработки очередей;
- влияет на безопасность;
- влияет на масштабирование;
- будет сложно изменить в будущем.

## Структура ADR

Каждый ADR должен содержать:

- номер;
- название;
- статус;
- дату;
- контекст;
- принятое решение;
- причины решения;
- последствия.

## Статусы

Возможные статусы:

- предложено;
- принято;
- отменено;
- заменено другим ADR.
