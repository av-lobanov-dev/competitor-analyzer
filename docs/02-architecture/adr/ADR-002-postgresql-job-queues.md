# ADR-002. Очереди заданий в PostgreSQL

- Статус: принято
- Дата: 2026-07-18

## Контекст

В проекте существуют задания исследования сайтов, создания правил парсинга и сбора товаров.

В дальнейшем несколько worker могут одновременно запрашивать новые задания. Необходимо исключить ситуацию, при которой два worker получают одно и то же задание.

## Решение

Очереди заданий хранятся в PostgreSQL.

Получение следующего задания выполняется атомарно с использованием SQL-конструкции:

```sql
FOR UPDATE SKIP LOCKED
```

При успешном получении задания его статус сразу изменяется:

```text
new -> running
```

## Требования к функции получения задания

Функция должна:

1. найти самое старое подходящее задание;
2. заблокировать выбранную строку;
3. пропустить строки, заблокированные другим worker;
4. изменить статус задания на `running`;
5. установить время начала обработки;
6. очистить старое сообщение об ошибке;
7. вернуть данные только захваченного задания.

## Причины

Решение позволяет:

- безопасно запускать несколько worker;
- не добавлять отдельную систему очередей на текущем этапе;
- сохранять задания и их статусы в одной базе;
- использовать транзакции PostgreSQL;
- видеть историю обработки.

## Последствия

Все новые очереди должны иметь согласованные статусы заданий.

Минимальный набор статусов:

- `new`;
- `running`;
- `completed`;
- `failed`.

Для каждой очереди должна существовать безопасная функция получения следующего задания.
