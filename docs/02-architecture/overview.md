# Общая архитектура

## Компоненты проекта

```text

   |
   v

   +-------------------+
   |                   |
   v                   v
Playwright            GPT
   |                   |
   +---------+---------+
             |
             v
        PostgreSQL
```

## Роли компонентов

### n8n

Управляет процессами проекта:

- запускает последовательности действий;
- работает с расписанием;
- создаёт задания;
- передаёт данные между компонентами;
- запускает уведомления и отчёты.

### Playwright

Выполняет действия на сайтах:

- открывает страницы;
- выполняет JavaScript;
- переходит по ссылкам;
- обрабатывает пагинацию;
- собирает структуру страниц;
- собирает товары и цены;
- записывает результаты в PostgreSQL.

### PostgreSQL

Является основным хранилищем проекта:

- хранит клиентов;
- хранит сайты конкурентов;
- хранит очереди заданий;
- хранит снимки страниц;
- хранит правила парсинга;
- хранит товары;
- хранит цены;
- хранит историю изменений;
- хранит результаты анализа.

### GPT

Выполняет аналитические задачи:

- анализирует структуру сайтов;
- создаёт правила парсинга;
- помогает сопоставлять одинаковые товары;
- анализирует цены и изменения;
- подготавливает выводы и отчёты.

## Архитектурные принципы

1. PostgreSQL является основным источником истины.
2. Основные задания хранятся в очередях PostgreSQL.
3. Worker должен безопасно получать только одно задание.
4. Для конкурентной обработки используется `FOR UPDATE SKIP LOCKED`.
5. Playwright собирает данные, но не принимает бизнес-решения.
6. GPT анализирует данные, но не выполняет непосредственный обход сайтов.
7. n8n управляет процессами, но не хранит основные бизнес-данные.
8. Техническая документация хранится в каталоге `docs`.
9. Изменения документации отслеживаются через Git.
