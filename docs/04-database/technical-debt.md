# Технический долг PostgreSQL

## Высокий приоритет

### Разделить базы n8n и приложения

Сейчас внутренние таблицы n8n и таблицы Competitor Analyzer находятся в одной базе и схеме.

Рекомендуемая структура:

```text
n8n
competitor_analyzer
```

или отдельные схемы:

```text
n8n.*
competitor_analyzer.*
```

Разделение по базам предпочтительнее.

### Добавить настоящий механизм миграций

Новые файлы из `postgres/init/` не применяются автоматически к существующей базе.

Нужен отдельный migration runner и таблица учёта миграций приложения.

### Исправить получение analysis_jobs

`take_next_analysis_job()` может забрать задание типа `scraping_rule`.

Необходимо фильтровать тип анализа или передавать его параметром.

### Настроить регулярные backup

Автоматических скриптов `pg_dump` сейчас нет.

Нужно создать:

- ежедневный backup;
- срок хранения;
- очистку старых копий;
- проверку восстановления;
- внешнее хранилище копий.

## Средний приоритет

### Добавить CHECK для scan_jobs.status

Допустимые значения:

```text
new
running
completed
failed
```

### Добавить NOT NULL к обязательным связям

Кандидаты:

```text
competitor_sites.client_id
scan_jobs.competitor_site_id
products.competitor_site_id
price_history.product_id
site_maps.competitor_site_id
```

Проверка показала, что текущих записей с `NULL` нет.

### Добавить уникальность товара

Product Worker считает товар уникальным по:

```text
competitor_site_id + url
```

Но база это не гарантирует.

Нужен уникальный индекс после проверки дубликатов.

### Добавить индексы на старые внешние ключи

Отдельных индексов нет у некоторых полей:

```text
competitor_sites.client_id
scan_jobs.competitor_site_id
products.competitor_site_id
price_history.product_id
site_maps.competitor_site_id
```

### Определить единое поведение удаления

Сейчас часть связей использует `CASCADE`, а часть `NO ACTION`.

Необходимо определить, что должно происходить с историей при удалении:

- клиента;
- сайта;
- правила;
- товара;
- задания.

## Низкий приоритет

### Перейти с SERIAL на IDENTITY

Новые таблицы можно создавать через:

```sql
GENERATED BY DEFAULT AS IDENTITY
```

### Хранить время с часовым поясом

Возможный стандарт:

```text
timestamp with time zone
```

или строгое соглашение, что все значения `timestamp without time zone` хранятся в UTC.

### Автоматически обновлять scraping_rules.updated_at

Сейчас поле имеет `DEFAULT NOW()`, но само по себе не меняется при UPDATE.

Нужен trigger или обновление поля в каждом SQL-запросе.
