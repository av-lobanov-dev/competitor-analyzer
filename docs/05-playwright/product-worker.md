# Worker сбора товаров

## Файл

```text
playwright/src/product-worker.js
```

## Назначение

Worker получает правило парсинга и собирает товары со страниц каталога конкурента.

## Источник заданий

Таблица:

```text
product_scan_jobs
```

Используются только задания:

```text
status = new
```

Правило должно быть активно:

```text
scraping_rules.is_active = true
```

Для безопасного получения задания используется:

```sql
FOR UPDATE OF psj SKIP LOCKED
```

## Изменение статусов

При получении:

```text
new -> running
```

При успехе:

```text
running -> completed
```

При ошибке:

```text
running -> failed
```

## Получаемые настройки правила

Worker получает:

- начальный URL;
- селектор карточки товара;
- селектор названия;
- селектор цены;
- селектор ссылки;
- селектор внешнего идентификатора;
- селектор старой цены;
- селектор валюты;
- селектор следующей страницы;
- максимальное количество страниц;
- валюту правила.

## Используемые поля

В текущей реализации используются:

- `start_url`;
- `product_card_selector`;
- `product_name_selector`;
- `product_price_selector`;
- `product_url_selector`;
- `product_external_id_selector`;
- `next_page_selector`;
- `max_pages`;
- `currency`.

Пока не используются:

- `product_old_price_selector`;
- `product_currency_selector`.

## Алгоритм сбора

1. Открывается начальная страница.
2. Проверяется HTTP-статус.
3. Ожидается селектор карточки товара.
4. Получается список карточек.
5. Для каждой карточки извлекаются данные.
6. Проверяется наличие обязательных полей.
7. Проверяется повторная ссылка.
8. Создаётся или обновляется товар.
9. Сохраняется запись цены.
10. Открывается следующая страница.
11. Цикл продолжается до достижения `max_pages`.

## Обязательные данные товара

Карточка сохраняется только при наличии:

- названия;
- корректно распознанной цены;
- ссылки товара.

При отсутствии одного из полей карточка пропускается.

## Поиск существующего товара

Товар ищется по сочетанию:

```text
competitor_site_id
url
```

Если товар найден:

- обновляется название;
- при наличии обновляется внешний идентификатор.

Если товар не найден:

- создаётся новая запись.

## Сохранение цены

Каждая успешно обработанная карточка создаёт запись в:

```text
price_history
```

Сохраняются:

- идентификатор товара;
- идентификатор задания;
- цена;
- валюта.

## Пагинация

Для перехода используется `next_page_selector`.

Worker завершает обход, если:

- селектор не задан;
- элемент следующей страницы не найден;
- у ссылки отсутствует `href`;
- достигнуто значение `max_pages`.

## Защита от повторов

В рамках одного задания используется набор обработанных URL.

Если один URL встретился повторно, карточка пропускается.

## Статистика задания

При завершении сохраняются:

- обработанные страницы;
- найденные карточки;
- созданные товары;
- обновлённые товары;
- сохранённые цены;
- пропущенные карточки.

## Обработка ошибок

При ошибке:

- задание получает статус `failed`;
- сохраняется сообщение и стек ошибки;
- длина ошибки ограничивается 10 000 символами;
- устанавливается время завершения.

## Известные ограничения

### Разбор цены

Текущая функция рассчитана в первую очередь на простой формат цены.

Сложные форматы могут распознаваться неправильно:

```text
12,990.00
1.299,50
```

### Старая цена

Старая цена пока не извлекается и не сохраняется.

### Валюта карточки

Валюта из самой карточки пока не извлекается.

Используется общая валюта из правила.

### Наличие товара

Наличие товара пока не собирается.

### Частичная ошибка

Ошибка одной карточки может остановить всё задание, если исключение не обработано внутри цикла карточек.

### Транзакции

Создание товара и сохранение цены выполняются отдельными запросами.

Они пока не объединены в одну транзакцию.
