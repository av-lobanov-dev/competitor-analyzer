# Обработчик очередей Playwright

## Файл

```text
playwright/src/queue-runner.sh
```

## Назначение

Скрипт постоянно работает внутри контейнера Playwright и периодически запускает worker.

Docker Compose запускает его командой:

```text
sh src/queue-runner.sh
```

## Интервал проверки

Интервал задаётся переменной:

```text
WORKER_INTERVAL_SECONDS
```

Если переменная отсутствует, используется значение:

```text
15 секунд
```

## Последовательность выполнения

В каждом цикле запускаются:

1. `node src/worker.js`;
2. `node src/product-worker.js`;
3. ожидание заданного интервала;
4. новый цикл.

## Обработка ошибок

Если один worker завершился с ошибкой, скрипт:

- записывает сообщение в журнал;
- не завершает контейнер;
- продолжает выполнение;
- переходит к следующему worker;
- через установленный интервал повторяет проверку.

## Ограничения

В текущем виде каждый worker получает максимум одно задание за один запуск.

Следовательно, один цикл может обработать:

- одно задание исследования сайта;
- одно задание сбора товаров.

Задания GPT из `analysis_jobs` этим скриптом пока не обрабатываются.

## Технический долг

В дальнейшем необходимо рассмотреть:

- отдельные процессы для разных worker;
- независимое масштабирование worker;
- контроль зависших заданий;
- повторные попытки;
- корректное завершение контейнера;
- отдельные журналы для каждого worker.
