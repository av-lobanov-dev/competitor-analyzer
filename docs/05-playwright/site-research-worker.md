# Worker исследования сайтов

## Файл

```text
playwright/src/worker.js
```

## Назначение

Worker получает новое задание исследования сайта, открывает страницу через Playwright и сохраняет снимок в PostgreSQL.

## Источник заданий

Таблица:

```text
scan_jobs
```

Выбираются задания со статусом:

```text
new
```

Задания обрабатываются по времени создания:

```text
ORDER BY created_at ASC
```

Для безопасного получения используется:

```sql
FOR UPDATE OF sj SKIP LOCKED
```

## Изменение статусов

При получении задания:

```text
new -> running
```

При успешном завершении:

```text
running -> completed
```

При ошибке:

```text
running -> failed
```

## Получаемые данные

Worker получает:

- идентификатор задания;
- идентификатор сайта конкурента;
- название конкурента;
- URL сайта.

## Настройки браузера

Используется Chromium в фоновом режиме:

```text
headless: true
```

Размер окна:

```text
1440 × 900
```

Тайм-аут открытия страницы:

```text
30 секунд
```

Ожидаемое событие загрузки:

```text
domcontentloaded
```

## Собираемые данные

Worker собирает:

- запрошенный URL;
- итоговый URL;
- заголовок страницы;
- текст страницы;
- HTML страницы;
- HTTP-статус;
- время загрузки;
- язык страницы;
- заголовки `h1`–`h6`;
- ссылки;
- кнопки;
- поля ввода;
- изображения;
- возможные карточки товаров;
- статистику найденных элементов.

## Ограничения размера

Текст страницы ограничивается:

```text
500 000 символов
```

HTML ограничивается:

```text
2 000 000 символов
```

## Ограничения количества элементов

Максимально сохраняется:

- 200 заголовков;
- 1 000 ссылок;
- 500 кнопок;
- 500 полей ввода;
- 1 000 изображений;
- 500 возможных карточек товаров.

## Сохранение результата

Снимок сохраняется в таблицу:

```text
page_snapshots
```

После сохранения создаётся задание анализа:

```text
analysis_type = site_structure
status = new
```

Задание создаётся в таблице:

```text
analysis_jobs
```

Для защиты от дублирования используется:

```sql
ON CONFLICT (page_snapshot_id, analysis_type)
DO NOTHING
```

## Завершение ресурсов

После обработки:

- браузер закрывается;
- соединения PostgreSQL закрываются через `pool.end()`.

## Ограничения текущей реализации

Worker исследует только стартовую страницу задания.

Полный обход внутренних страниц сайта в актуальном `worker.js` пока не реализован.

Возможные карточки товаров определяются эвристически по:

- микроразметке Product;
- атрибутам `data-product`;
- классам, содержащим `product`;
- классам, содержащим `card`;
- наличию ссылки или текста цены.
