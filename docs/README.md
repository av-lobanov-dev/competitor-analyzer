## Правило обновления документации

После завершения каждого этапа необходимо:

1. Обновить описание изменённого компонента.
2. Обновить `current-state.md`.
3. Обновить `roadmap.md`, если изменился статус этапов.
4. Добавить запись в `changelog.md`.
5. Создать ADR, если принято важное архитектурное решение.
6. Зафиксировать документацию в Git отдельным коммитом или вместе с соответствующим изменением кода.
