# Безопасный доступ к n8n

## Текущее состояние

Веб-интерфейс n8n не публикуется напрямую в интернет.

В `docker-compose.yml` используется локальная привязка:

```yaml
ports:
  - "127.0.0.1:5678:5678"
```

Это означает:

- n8n доступен на сервере по адресу `http://127.0.0.1:5678`;
- порт `5678` недоступен напрямую через публичный IP сервера;
- доступ с рабочего компьютера выполняется через SSH-туннель.

## Подключение с macOS

Открыть отдельный терминал на Mac и выполнить:

```bash
ssh -N -o ExitOnForwardFailure=yes -L 15678:127.0.0.1:5678 root@SERVER_IP
```

После подключения открыть:

```text
http://localhost:15678
```

Пока туннель работает, команда SSH не возвращает приглашение терминала. Это нормально.

Для остановки туннеля нажать `Ctrl+C`.

## Проверка на сервере

Проверить публикацию порта:

```bash
docker port competitor_n8n 5678
```

Ожидаемый результат:

```text
127.0.0.1:5678
```

Проверить сокет:

```bash
ss -lntp | grep ':5678 '
```

Проверить локальный ответ:

```bash
curl --silent --output /dev/null --write-out 'HTTP status: %{http_code}\n' --max-time 10 http://127.0.0.1:5678/
```

Ожидаемый результат:

```text
HTTP status: 200
```

## Проверка внешнего доступа

Прямой адрес:

```text
http://SERVER_IP:5678
```

не должен открываться.

Доступ через SSH-туннель должен работать:

```text
http://localhost:15678
```

## Диагностика

Проверить разрешение SSH-туннелей:

```bash
sshd -T | grep -E '^(allowtcpforwarding|disableforwarding|permitopen) '
```

Ожидаемые значения:

```text
allowtcpforwarding yes
disableforwarding no
permitopen any
```

Проверить контейнер:

```bash
docker compose ps n8n
```

Посмотреть последние логи:

```bash
docker logs --tail 150 competitor_n8n
```

## Требования безопасности

Не следует возвращать публикацию:

```yaml
ports:
  - "5678:5678"
```

Она открывает n8n на всех сетевых интерфейсах сервера.

До внедрения HTTPS и обратного прокси рекомендуемый способ доступа — SSH-туннель.
