## Текущий приоритет

Перед выпуском v0.1 необходимо:

1. закрыть внешний доступ к PostgreSQL;
2. защитить n8n;
3. удалить секреты из исходного кода;
4. добавить swap;
5. внедрить healthcheck;
6. внедрить контролируемые миграции PostgreSQL.
