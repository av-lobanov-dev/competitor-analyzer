# Память сервера и swap

## Текущая конфигурация

На VDS установлено около 2 ГБ оперативной памяти.

20 июля 2026 года был добавлен swap-файл размером 2 ГБ:

```text
/swapfile
```

Права доступа к swap-файлу:

```text
600
```

Swap автоматически включается после перезагрузки сервера через `/etc/fstab`:

```text
/swapfile none swap sw 0 0
```

## Параметры ядра

Настройки памяти хранятся в файле:

```text
/etc/sysctl.d/99-competitor-analyzer-memory.conf
```

Используются следующие значения:

```text
vm.swappiness=10
vm.vfs_cache_pressure=50
```

Параметр `vm.swappiness=10` уменьшает склонность Linux заранее переносить данные из оперативной памяти в swap.

Параметр `vm.vfs_cache_pressure=50` позволяет ядру дольше сохранять кэш файловой системы.

## Проверка памяти

Проверить использование оперативной памяти и swap:

```bash
free -h
```

Проверить активные swap-устройства:

```bash
swapon --show
```

Проверить параметры ядра:

```bash
sysctl vm.swappiness
sysctl vm.vfs_cache_pressure
```

Проверить использование ресурсов контейнерами:

```bash
docker stats --no-stream
```

## Проверка после перезагрузки

После перезагрузки сервера необходимо выполнить:

```bash
free -h
swapon --show
sysctl vm.swappiness
sysctl vm.vfs_cache_pressure
docker ps
```

Ожидаемый результат:

- swap-файл `/swapfile` активен;
- размер swap составляет около 2 ГБ;
- `vm.swappiness` равен `10`;
- `vm.vfs_cache_pressure` равен `50`;
- контейнеры PostgreSQL, n8n и Playwright запущены.

## Важные замечания

Swap снижает риск аварийного завершения процессов при кратковременной нехватке оперативной памяти, но не заменяет оперативную память.

Если Playwright, n8n или PostgreSQL регулярно используют swap, необходимо:

1. уменьшить параллелизм Playwright;
2. установить ограничения памяти контейнеров;
3. проверить процессы на утечки памяти;
4. увеличить объём оперативной памяти VDS.
