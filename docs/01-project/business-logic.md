# Бизнес-логика

## Общая последовательность

1. Клиент предоставляет список сайтов конкурентов.
2. Для каждого сайта создаётся задание исследования.
3. Playwright открывает сайт и собирает снимки страниц.
4. Данные страниц сохраняются в PostgreSQL.
5. GPT анализирует сохранённые данные.
6. GPT формирует правило сбора товаров.
7. Правило сохраняется в PostgreSQL.
8. Создаётся задание на сбор товаров.
9. Playwright собирает товары по сохранённому правилу.
10. Результаты сохраняются в PostgreSQL.
11. Система фиксирует изменения цен и наличия.
12. GPT анализирует данные и подготавливает результат для клиента.

## Ответственность компонентов

### n8n

Оркестратор процессов:

- запускает последовательности действий;
- управляет расписанием;
- передаёт данные между компонентами;
- запускает обработку и уведомления.

### Playwright

Исполнитель интернет-задач:

- открывает сайты;
- выполняет JavaScript страниц;
- переходит по ссылкам и пагинации;
- собирает структуру страниц и товары;
- записывает результаты в PostgreSQL.

### PostgreSQL

Основное хранилище:

- клиенты;
- сайты конкурентов;
- задания;
- снимки страниц;
- правила парсинга;
- товары;
- цены;
- история изменений;
- результаты анализа.

### GPT

Аналитический компонент:

- анализирует структуру сайта;
- создаёт правила парсинга;
- помогает сопоставлять товары;
- анализирует изменения;
- подготавливает выводы и отчёты.

## Разделение ответственности

Playwright собирает данные, но не принимает бизнес-решения.

GPT анализирует данные, но не выполняет непосредственный обход сайтов.

n8n управляет процессом, но не хранит основные бизнес-данные.

PostgreSQL является основным источником данных проекта.
